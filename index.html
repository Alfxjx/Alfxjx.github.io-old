Hexo主题设置缩略图

2019-02-22

给自己的博客加上了Material-T主题，但是安装上配置了一番之后，发现好看的文章缩略图无法显示了，当然当你读到这篇文章的时候，已经解决了，以下是我的解决过程：

  1. 首先，根据文档的内容，在文章开头的front matter里面添加 了图片，发现缩略图不显示，但是会显示一个占位的图，当时是在vscode里面编辑的，在Material-T/layout/index.ejs文件中出现了一个报错，研究不明白于是去项目的github上提了一个issue。

  2. 后来发现这个报错是一个假报错，换到Webstorm就不显示了。但是事实上的问题还是没解决，于是又询问了作者。

  3. 今天作者回复我说：

    涉及到的 Hexo 解析逻辑是这样的：

    1. 从 /source/_posts 中读取文章内容渲染成HTML；
    2. 将 /themes/Material-T/source/ 下的文件全部拷贝到网站根目录；

    所以建议所有图片引用使用相对于根目录的 url ，比如我把图片放到 /themes/Material-T/source/here_is_my_img/ ，引用的时候链接应为 /here_is_my_img/***.jpg。

于是如法炮制,发现实际上应该是保存在/source/post_img/中，和主题无关。

这。。。

就这样吧， issue都被我关掉了。

PS：

Hexo在每一次内容更改过后需要做的操作：

hexo clean ——清理文件
hexo generate / hexo g ——生成静态文件
hexo server / hexo s ——启动本地服务器
hexo deploy / hexo d ——部署网站
