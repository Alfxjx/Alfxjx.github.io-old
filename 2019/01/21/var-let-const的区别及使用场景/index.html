var,let,const的区别及使用场景

Jan 21 2019

var & let & const

  • var 变量，挂载在window全局作用域上
  • let 和 const 是ES6语法，表示局部变量和常量。

使用场景

建议使用的优先级：const > let > var

ES6 提出了两个新的声明变量的命令：let和const。其中，let完全可以取代var，因为两者语义相同，而且let没有副作用。

var命令存在变量提升效用，let命令没有这个问题。

const定义的变量不可以修改，而且必须初始化。

let是块级作用域，函数内部使用let定义后，对函数外部无影响。同一个左右域不要重复定义汇报错，同时funB无法访问到a。

在let和const之间，建议优先使用const，尤其是在全局环境，不应该设置变量，只应设置常量。
